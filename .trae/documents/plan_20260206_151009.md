Para implementar a nova estrutura do painel administrativo e a funcionalidade de horário de funcionamento, seguirei os seguintes passos:

### 1. Banco de Dados (Migração)
- Criar uma nova tabela `store_settings` no Supabase para armazenar as configurações da loja.
- Campos sugeridos: `id`, `opening_hours_start` (hora de abertura), `opening_hours_end` (hora de fechamento), `is_open` (flag manual se necessário), `updated_at`.
- Inserir um registro padrão inicial.

### 2. Nova Página de Configurações (`AdminSettings.tsx`)
- Criar o componente `AdminSettings` para gerenciar o horário de funcionamento.
- Implementar campos de input para horário de abertura e fechamento.
- Adicionar botão para salvar as alterações no Supabase.

### 3. Reestruturação do Painel Admin (`AdminPage.tsx`)
- Atualizar o estado `activeTab` para suportar as novas abas: `'orders'`, `'products'`, `'shipping'`, `'settings'`.
- Remover a aba 'dashboard' (conforme solicitado, as opções devem ser iguais às abas).
- Reorganizar a ordem das abas: **Pedidos**, **Produtos**, **Ajuste de Frete**, **Configurações**.
- Atualizar os Cards de navegação rápida para refletir essa mesma estrutura e direcionar para as respectivas abas.
- Integrar o componente `AdminSettings` na aba de Configurações.
- Integrar o `ShippingRatesManager` como uma aba principal (antes era um modal).

### 4. Exibição no Carrinho (`CartPage.tsx`)
- Buscar as configurações da loja (`store_settings`) ao carregar o carrinho.
- Verificar se o horário atual está dentro do intervalo configurado.
- Exibir uma mensagem informativa acima do botão "Finalizar Pedido" com o horário de funcionamento, sem bloquear a compra.

### 5. Atualização de Rotas (Opcional/Verificação)
- O usuário mencionou `http://localhost:8080/configuracoes`. Se desejar uma rota dedicada, precisarei atualizar o `App.tsx` e o `main.tsx` para incluir essa rota, mas a integração dentro do `AdminPage` via abas (como SPA) costuma ser mais fluida. Vou manter como uma aba dentro do Admin para consistência, mas se o requisito for estritamente uma URL separada, posso criar a rota `/admin/settings`. **Assumirei a estrutura de abas dentro do AdminPage para manter o padrão atual, mas com a URL refletindo a aba se possível ou apenas navegação interna.**
